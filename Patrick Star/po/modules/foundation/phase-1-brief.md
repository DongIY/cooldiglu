# Phase 1 Brief

## 当前判断

这是一个从零启动的 Godot 校园恋爱日常 RPG 项目。
本轮首要目标不是堆内容，而是先搭出 **可反复迭代的最小可玩闭环**。

## 竞品共性提炼

### 事实

- 公开玩法资料显示，此类作品常见主循环为：**地点切换 → 点位调查 / NPC互动 → 时段推进 → 关系或进度变化 → 解锁新事件 / 图像奖励**。
- 好感度或关系阈值通常承担内容闸门作用，用来控制角色线推进速度。
- 插画 / 图像奖励不是孤立系统，而是作为剧情推进后的可视化反馈和收藏驱动。

### 推断

- 对于小体量独立项目，最容易失控的部分不是地图或代码，而是角色线和事件数量。
- 如果首版同时做多地图、多角色、多资源系统，极易在内容产能与回归测试上失控。

### 待验证

- 后续是否需要加入更多日程管理、道具、兼职或小游戏循环，目前暂不纳入首版。

## 本轮目标

- 锁定首版 MVP 边界
- 产出基础设计简案与 QA 清单
- 初始化 Godot 工程与 GitHub 协作骨架
- 跑通一个可演示的探索事件闭环

## 本轮不做

- 不做复杂战斗系统
- 不做多地图切换网络式扩张
- 不做大型任务树或经营系统
- 不做正式美术资源整包导入

## 首版 MVP 定义

### 地图

- 1 张校园主地图
- 4 个关键互动点：校门、教学楼前、操场边、天台入口

### 角色

- 2 名核心角色
  - `lin`：学生会干部，偏认真路线
  - `yuki`：摄影社成员，偏轻松路线

### 系统

- 玩家移动与交互提示
- 时段推进（早晨 / 放学后 / 夜晚）
- 基础对话与选项
- 好感度增减与阈值判断
- 事件触发与标记写入
- CG 解锁与图鉴展示
- 快速存档 / 读档

## 验收口径

达到以下条件即可视为首版闭环成立：

1. 玩家可在校园主地图自由移动
2. 进入互动点后能触发对应事件
3. 至少 1 条角色线可以通过多次互动推进到结尾事件
4. 结尾事件能解锁 1 张 CG 并在图鉴中展示
5. 存档后重新读取，日期、时段、好感度、已解锁 CG 均保持一致
